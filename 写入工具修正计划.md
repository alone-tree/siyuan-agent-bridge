# 写入工具修正计划

本文面向后续执行实现的 AI，目标是修正 `siyuan_create_document` / `siyuan_edit_document` 初步测试后确认需要本轮处理的两个问题。实现前先阅读 `AGENTS.md`、`PRO.md`、`思源API.md` 和当前代码。

## 本轮目标

1. 移除快照 `tags` / `path` 参数，`createSnapshot` 只发送 `memo`。
2. 创建文档时避免正文首个同名 H1 与思源文档标题重复。

## 改动一：快照只发送 memo

思源 `/api/repo/createSnapshot` 当前源码只解析 `memo`，不支持 `path` 或 `tags`。该 API 创建的是整个工作空间/数据仓库快照，不是单文档或局部路径快照。

修改要求：

1. `SiYuanClient.create_snapshot()` 改为只接受 `memo: str`。
2. payload 只发送：

```python
{"memo": memo}
```

3. 删除 `tags`、`path` 参数和相关 payload 处理。
4. 所有调用点改为：

```python
client.create_snapshot(memo)
```

5. 更新测试，删除对 tags/path payload 的断言。

memo 格式同步调整为：

```text
siyuan-agent-bridge:auto-snapshot tool=<tool> target=<title-or-path> created=<yyyymmddHHMMSS>
```

时间使用本机本地时间，例如 `20260101123546`。

## 改动二：创建文档时避免重复 H1

产品语义：

- 思源文档标题属于文档树标题。
- Markdown 正文里的一级标题属于正文大纲。
- 创建文档时，如果正文第一个一级标题和 `title` 完全相同，应自动移除这个 H1。
- 如果正文第一个 H1 与 `title` 不同，保留它。

规范化规则：

```text
如果 markdown 的第一个非空行是 "# <heading>"
并且 heading.strip() == title.strip()
则删除这一行。
否则不改 markdown。
```

实现建议：

```python
def normalize_new_document_markdown(title: str, markdown: str) -> str:
    ...
```

要求：

- 只处理第一个非空行。
- 只处理 ATX H1：`# title`。
- 不处理 `##`、Setext heading、带复杂属性的标题。
- 移除 H1 后如果正文为空，保持原有 `markdown is required` 拒绝语义。
- 保持实现简单。

MCP 工具描述或 Skill 提示中补一句：创建文档时，正文默认不需要再写与文档标题相同的一级标题，除非用户明确要求不同的正文大纲标题。

## 测试计划

单元测试：

1. `create_snapshot()` 只发送 `{"memo": memo}`。
2. 写入工具调用 `create_snapshot(memo)` 时不传 tags/path。
3. `normalize_new_document_markdown()`：
   - 首个 H1 等于 title → 删除。
   - 首个 H1 不等于 title → 保留。
   - 前面有空行，之后 H1 等于 title → 删除。
   - `## title` → 不处理。
   - 删除 H1 后正文为空 → 创建工具拒绝空 markdown。

MCP/集成测试：

1. 创建文档，markdown 带同名 H1。
2. 读取文档，确认不出现重复 H1。
3. 创建文档，markdown 带不同 H1。
4. 读取文档，确认不同 H1 保留。
5. 查询快照列表，确认快照 memo 存在且 tag 为空。

## 验收标准

- 自动快照不再传 `tags` 或 `path`。
- 快照列表中本工具新建快照的 tag 为空。
- `siyuan_create_document` 不再因同名 H1 产生重复一级标题。
- 不同于 title 的正文 H1 不被误删。
- 单元测试通过。
- 文档说明与实现一致。
