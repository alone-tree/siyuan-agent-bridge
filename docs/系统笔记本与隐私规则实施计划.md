# 系统笔记本与隐私规则实施计划

## 背景

当前 `AI Guide`、`Workspace Index` 和隐私规则仍主要依赖项目本地文件或 MCP 工具维护。这个方案在单工作空间、开发者自用场景可用，但无法很好地跟随思源工作空间切换，也不够适合普通用户维护隐私规则。

目标是把工作空间级配置迁移到思源内部的系统笔记本中，让每个思源工作空间自带：

1. AI 使用规则。
2. AI 语义导航索引。
3. 给人的工具说明和开发者消息。
4. 人类维护的隐私隐藏规则。

详细设计依据见 `docs/PRO.md` 中“问题 11：系统笔记本 — Guide / Index 的归属”。

## 总目标

实现一个系统笔记本机制：

```text
中文环境：
思源代理桥
├── AI使用指南
├── 工作空间索引
├── 关于思源代理桥
└── 隐私规则

英文环境：
SiYuan Agent Bridge
├── AI Guide
├── Workspace Index
├── About SiYuan Agent Bridge
└── Privacy Rules
```

同一个工作空间只维护一套系统笔记本和系统文档。查找时兼容中英文名称；只有不存在时，才根据当前语言创建。

## 不做的事情

1. 不新增 `siyuan_write_index` MCP 工具。
2. 不同时创建中英文两套系统笔记本。
3. 不自动重命名已有系统笔记本或系统文档。
4. 不让 AI 读取、搜索、总结或编辑隐私规则文档。
5. 不使用 JSON 作为普通用户编辑隐私规则的主格式。
6. 不在第一阶段实现复杂多语言框架；只做轻量 `zh-CN` / `en` 模板和可扩展映射。

## 语言策略

### 语言选择

语言解析建议封装为独立函数，例如：

```python
resolve_language(config) -> str
```

优先级：

1. 显式配置，例如 `language = "zh-CN"` 或 `language = "en"`。
2. 系统 locale。
3. 已存在系统笔记本/系统文档的语言。
4. 检测失败默认 `zh-CN`，但要提示“语言检测失败，默认使用中文”。

第一阶段只需要支持：

```text
zh-CN
en
```

其他语言统一降级到 `en`，但内部结构要方便以后增加 `fr`、`de` 等语言。

### 名称映射

内部逻辑必须使用稳定 key，不直接依赖显示名称：

```python
SYSTEM_DOC_KEYS = [
    "ai_guide",
    "workspace_index",
    "about",
    "privacy_rules",
]
```

建议映射：

```python
SYSTEM_NOTEBOOK_NAMES = {
    "zh-CN": "思源代理桥",
    "en": "SiYuan Agent Bridge",
}

SYSTEM_DOC_NAMES = {
    "ai_guide": {
        "zh-CN": "AI 使用指南",
        "en": "AI Guide",
    },
    "workspace_index": {
        "zh-CN": "工作空间索引",
        "en": "Workspace Index",
    },
    "about": {
        "zh-CN": "关于 SiYuan Agent Bridge",
        "en": "About SiYuan Agent Bridge",
    },
    "privacy_rules": {
        "zh-CN": "隐私规则",
        "en": "Privacy Rules",
    },
}
```

查找规则：

1. 查找系统笔记本时，遍历所有已知语言的系统笔记本名称。
2. 找到任意一个就使用现有笔记本。
3. 找不到才按当前语言创建。
4. 查找系统文档时，对每个内部 key 遍历所有已知语言名称。
5. 找到任意一个就使用现有文档。
6. 找不到才按当前语言创建需要自动创建的文档。

不要因为用户切换系统语言而自动新建或重命名。特别提示，无论用户打开还是关闭该笔记本，都不影响MCP和skill。因为某些用户可能长期让AI自动维护index和guide，关掉笔记本可以减少视觉占用空间。

## 系统笔记本服务层

建议新增模块：

```text
source_code/agent_notebook.py
```

职责：

1. 定位或创建系统笔记本。
2. 定位系统文档。
3. 创建默认 `AI 使用指南 / AI Guide`。
4. 创建或更新 `关于思源代理桥 / About SiYuan Agent Bridge`。
5. 创建并解析 `隐私规则 / Privacy Rules`。如果解析失败报错，让AI提示人类修复文档，修复不好可以删掉文档或者整个笔记本，再重新启动Skill/MCP。因为重启后会自动创建相应文档。
6. 读取 `工作空间索引 / Workspace Index`，不存在时返回空状态。

建议提供的数据结构：

```python
@dataclass
class AgentNotebookState:
    language: str
    notebook_id: str
    notebook_name: str
    ai_guide_doc_id: str
    ai_guide_markdown: str
    workspace_index_doc_id: str | None
    workspace_index_markdown: str | None
    about_doc_id: str
    privacy_rules_doc_id: str
    privacy_rules: PrivacyRules
```

建议核心函数：

```python
ensure_agent_notebook(client, root, config) -> AgentNotebookState
```

`siyuan_start` 和 `siyuan_refresh_index` 都应调用它。

## Client API

需要在 `source_code/client.py` 中补充必要 API。

### 创建笔记本

新增：

```python
def create_notebook(self, name: str) -> dict[str, Any]:
    ...
```

调用：

```text
/api/notebook/createNotebook
payload: {"name": name}
```

返回值示例：

```json
{
  "notebook": {
    "id": "...",
    "name": "...",
    "closed": false
  }
}
```

### 文档定位

优先复用已有 SQL 能力。可以通过 blocks 表定位系统笔记本下的文档：

```sql
SELECT id, box, hpath, content
FROM blocks
WHERE type = 'd' AND box = '<notebook_id>'
```

匹配 hpath 的最后一段或完整路径：

```text
/AI 使用指南
/AI Guide
```

第一阶段系统文档都放在系统笔记本根目录，不做子目录。

## 系统文档创建规则

### AI Guide

不存在则创建；存在则不覆盖。

中文默认模板应精简，说明这是给 AI 的规则文档，用户可以写：

```text
- 偏好的回复风格
- 重点笔记本
- 不希望 AI 做的事情
- 写作和研究偏好
```

英文模板同理。

### Workspace Index

不自动创建。

如果不存在：

1. `siyuan_start` 启动包提示尚未创建。
2. Skill 引导 AI 在用户确认后创建。
3. 创建和更新复用现有 `siyuan_create_document` / `siyuan_edit_document`。

不要新增 `siyuan_write_index`。

### About SiYuan Agent Bridge

不存在则创建。

存在时通过模板版本标识判断是否更新。建议模板开头包含：

```markdown
<!-- siyuan-agent-bridge:about-template-version=1 -->
```

如果现有文档包含当前版本标识，跳过。

如果不包含当前版本标识，可以覆盖。该文档是系统维护的开发者消息渠道，应在正文明确提醒用户不要记录个人内容。

### Privacy Rules

不存在则创建默认表格。

存在后不覆盖，只解析。

它是 MCP server 的内部配置，不是 AI 可读知识库文档。

## Privacy Rules 模板

### 中文模板

```markdown
# 隐私规则

隐私规则完全由人类在本文档控制，AI 无法阅读/编辑/删除该文档。

请只编辑下面两张表格。你可以新增或删除表格行，但不要新增表格，也不要编辑表头，否则会报错。`Hide` 填 `yes` 才会对 AI 隐藏；填 `no` 表示暂不启用。你可以把某一行临时改为 `no` 来短暂开放给 AI，交流完毕后再改回 `yes`。文档修改会在每次工具刷新时生效，你可以告诉 AI“刷新一下”。

隐藏文档必须填写文档 ID。标题只给你自己确认，系统不会按标题匹配，因为文档标题容易重复。

隐藏笔记本时优先填写笔记本 ID。获取方法：在笔记本列表中点击笔记本右侧三个点，选择“设置”，然后点击“复制 ID”。如果暂时不知道 ID，也可以只填写笔记本名称；若多个笔记本重名，同名笔记本都会被隐藏。

## 隐藏笔记本

| Hide | 笔记本ID（建议填） | 笔记本名称 | 备注（可选） |
|-------------|------------------|------------|--------------|
| no | 20260503123456-abcdefg | 示例笔记本 | 示例，不会生效 |
| yes | 20260503123456-abcdefg | 示例：私人资料 | 示例，会隐藏 |
| yes |  | 示例：个人日记 | 按名称隐藏 |

## 隐藏文档

| Hide | 文档ID（必填，不填会报错） | 标题（仅供确认） | 备注（可选） |
|-------------|----------------|------------------|--------------|
| no | 20260503123456-abcdefg | 示例文档 | 示例，不会生效 |
| yes | 20260503123456-abcdefg | 示例：未公开项目 | 示例，会隐藏 |
```

### 英文模板

```markdown
# Privacy Rules

This document controls which notes are hidden from AI. AI cannot read this document.

Only edit the two tables below. You may add or remove rows, but do not add tables or edit table headers. A row hides content from AI only when `Hide` is `yes`; `no` means disabled.

Document hiding requires Document ID. Title is only for your confirmation and is not used for matching.

For notebooks, Notebook ID is preferred. To get it, click the three-dot menu next to the notebook, open Settings, and click Copy ID. If you do not know the ID yet, use Notebook Name. If multiple notebooks share the same name, all matching notebooks will be hidden.

## Hide Notebooks

| Hide | Notebook ID | Notebook Name | Reason |
|---------------|-------------------------|---------------|-----------------|
| no | 20260503123456-abcdefg | Example: Private Notebook | Example, ignored |
| yes | 20260503123456-abcdefg | Example: Private Data | Example, hidden |

## Hide Documents

| Hide | Document ID | Title | Reason |
|---------------|------------------------|-------------------------------|-----------------|
| no | 20260503123456-abcdefg | Example: Private Project | Example, ignored |
| yes | 20260503123456-abcdefg | Example: Unpublished Project | Example, hidden |
```

## Privacy Rules 解析

### 表格定位

中文标题：

```text
## 隐藏笔记本
## 隐藏文档
```

英文标题：

```text
## Hide Notebooks
## Hide Documents
```

只解析这些固定标题下的第一张 Markdown 表格。

### 表头别名

解析器应使用内部字段 key，并支持中文/英文表头别名。如果报错应该返回可定位的错误信息，以便用户排查修改，不要只单纯地说“隐私规则解析失败”。

表格标题和表头应尽量按模板填写，但解析器应做温和兼容，避免因为空格、括号或旧模板字段造成不必要的失败。兼容原则是“尽量不给用户造成麻烦”，但不能把自然语言段落当作规则解析，也不能支持模糊匹配、正则或路径隐藏。

笔记本表：

```text
active:
  - Hide
  - Enabled
  - 对AI隐藏？
  - 对 AI 隐藏？
  - Hide from AI?

notebook_id:
  - 笔记本ID（建议填）
  - 笔记本 ID（建议填）
  - 笔记本ID（优先）
  - 笔记本 ID（优先）
  - Notebook ID (preferred)
  - Notebook ID

notebook_name:
  - 笔记本名称
  - Notebook Name

reason:
  - 备注（可选）
  - 备注
  - Note (optional)
  - Reason
```

文档表：

```text
active:
  - Hide
  - Enabled
  - 对AI隐藏？
  - 对 AI 隐藏？
  - Hide from AI?

document_id:
  - 文档ID（必填，不填会报错）
  - 文档 ID（必填，不填会报错）
  - 文档ID（必填）
  - 文档 ID（必填）
  - Document ID (required)
  - Document ID

title:
  - 标题（仅供确认）
  - 标题
  - Title (for confirmation only)
  - Title

reason:
  - 备注（可选）
  - 备注
  - Note (optional)
  - Reason
```

### 激活值

模板推荐：

```text
推荐：yes / no
兼容：是 / 否
```

解析器可接受：

```text
active: 是, yes, true, 1
inactive: 否, no, false, 0, 空字符串
```

遇到其他值视为解析失败。

### 匹配规则

笔记本隐藏：

1. 如果填写 `Notebook ID`，按 ID 精确匹配。
2. 如果没有 ID，按 `Notebook Name` 精确匹配。
3. 如果多个笔记本名称相同，全部隐藏。
4. 如果 ID 不存在，解析失败。
5. 如果只写名称但没有任何匹配，解析失败。

文档隐藏：

1. 必须填写 `Document ID`。
2. `Title` 只给人确认，不参与匹配。
3. ID 不存在时解析失败。
4. 隐藏文档等于隐藏该文档及所有子文档。

备注：

1. `Reason` / `备注` 只给人审计。
2. 不参与匹配。

### 失败策略

隐私规则解析失败时，不能退化为“没有隐藏规则”。不需要把隐私检查状态落盘成持久文件。第一阶段每次 start / refresh / search / read / write 操作前解析 `Privacy Rules`：解析成功则本次操作继续，解析失败则本次操作中止。后续如果需要性能优化，可以只做进程内短期缓存。

应中止：

```text
siyuan_start
siyuan_refresh_index
siyuan_find_documents
siyuan_read_document
siyuan_create_document
siyuan_edit_document
```

返回给 AI 的错误信息不得包含具体隐藏笔记本名称、文档 ID 或标题，但应该包括可定位的错误信息。允许返回表格名、行号、字段名和错误类型；不返回该字段里的具体值。

```text
隐私规则格式错误：
- Hide Documents 第 3 行：Document ID 为空。
- Hide Notebooks 第 2 行：Hide 只能填写 yes/no。
- Hide Documents 第 4 行：Document ID 不存在或不可访问。
```

不要返回具体隐藏的笔记本名称、文档标题或 ID。

## 与现有 ignore 规则的关系

现有文件：

```text
siyuan.ignore.local.json
siyuan.allow.local.json
```

当前仍处于开发阶段，没有外部用户需要兼容。系统笔记本上线后，不再做 `siyuan.ignore.local.json` / `siyuan.allow.local.json` fallback，也不合并旧规则。隐私规则主副本就是思源系统笔记本中的 `Privacy Rules` / `隐私规则`。

旧隐私 MCP 工具可以暂时保留开发兼容，但 Skill 和 README 不再引导 AI 使用它们；后续稳定后可以删除。

## AI 可见性规则

`Privacy Rules` 必须硬隔离：

1. 不进入 `docs.jsonl`。
2. 不进入 `tree.md`。
3. 不被 `siyuan_list` 展示。
4. 不被 `siyuan_find_documents` 返回。
5. 不被 `siyuan_read_document` 读取。
6. 不进入 `siyuan_start` 启动包。
7. 不允许 `siyuan_edit_document` / `siyuan_create_document` 由 AI 修改它。

系统笔记本中的其他文档不强隔离：

1. `AI Guide` 和 `Workspace Index` 会进入启动包。
2. `About` 只在启动包中提及路径和用途，不塞全文。
3. 普通索引/搜索可看到系统笔记本其他文档，但 Skill 必须要求 AI 不把它们作为用户原始资料。

## 启动包调整

`siyuan_start` 应返回：

1. 语言偏好。
2. 笔记本概览。
3. `Workspace Index` 内容，如果存在。
4. `AI Guide` 内容。
5. `About` 文档路径和用途的一行说明。
6. `Privacy Rules` 加载状态，但不包含规则内容。

语言偏好示例：

```markdown
## 语言偏好

检测到的用户/工作空间语言：zh-CN
优先回复语言：中文
除非用户明确要求使用其他语言，否则默认用中文回复。
```

英文：

```markdown
## Language Preference

Detected user/workspace language: en
Preferred reply language: English
Use English by default when talking to the user, unless the user explicitly asks for another language.
```

隐私状态示例：

```text
隐私规则：已加载，2 条笔记本规则，1条生效。3 条文档规则，2条生效。
```

不要列出具体规则。

## 隐藏规则保护

系统笔记本不能被隐藏。

如果用户尝试隐藏：

```text
思源代理桥 / SiYuan Agent Bridge
```

或其下任一系统文档/子文档，MCP 应拒绝并提示：

```text
这是系统笔记本，隐藏后启动包和隐私规则会失效。如果确实有私密内容，请先移动到其他笔记本再隐藏。
```

这个保护应作用于：

1. 旧的 `siyuan_privacy` 工具。
2. 新的 `Privacy Rules` 解析结果。

也就是说，用户不能在 `Privacy Rules` 表格里把系统笔记本或系统文档隐藏掉。（但需要注意和隐私规则的兼容性，不要冲突）

## 实现步骤

### 1. 新增语言模块

可新增：

```text
source_code/i18n.py
```

内容：

1. 语言解析。
2. 系统笔记本名称映射。
3. 系统文档名称映射。
4. 默认模板。

保持简单，不引入 gettext。

### 2. 扩展 Client

在 `client.py` 增加：

```python
create_notebook(name)
```

如现有代码缺少文档按路径查找能力，可用 SQL 辅助查找系统文档。

### 3. 新增 agent_notebook 服务层

实现：

```python
ensure_agent_notebook(client, root, config)
```

内部完成：

1. 找/建系统笔记本。
2. 找/建 AI Guide。
3. 找/建 About。
4. 找/建并解析 Privacy Rules。
5. 查找 Workspace Index。

### 4. 接入 siyuan_start

修改 `mcp_server.py`：

1. `siyuan_start` 调用 `ensure_agent_notebook`。
2. 启动包改为读取思源系统笔记本里的 AI Guide / Workspace Index。
3. 添加语言偏好。
4. 添加 About 路径说明。
5. 添加 Privacy Rules 加载状态。

### 5. 接入 refresh/index/search/read

确保：

1. refresh 前解析 Privacy Rules。
2. indexer 使用解析后的隐私规则过滤。
3. search/read 使用同一规则过滤。
4. Privacy Rules 文档本身硬隔离，不返回给 AI。

### 6. 调整写入保护

确保现有写入工具：

1. 不允许 AI 编辑 `Privacy Rules`。
2. 不允许 AI 隐藏或修改系统笔记本的安全边界。
3. 仍允许 AI 在用户确认后创建/更新 Workspace Index。

### 7. 更新 Skill

更新 Skill 指令：

1. 启动后遵循语言偏好回复用户。
2. 创建索引时写入系统笔记本的工作空间索引。

## 给执行 AI 的提示词

你要在 `D:\Github\siyuan-agent-bridge` 中实现系统笔记本与 Privacy Rules 表格隐私规则。请先阅读 `docs/PRO.md` 的“问题 11：系统笔记本 — Guide / Index 的归属”和本计划文件。目标是在思源中自动维护一个系统笔记本，中文环境名为 `思源代理桥`，英文环境名为 `SiYuan Agent Bridge`；查找时兼容中英文名称，已有则复用，不自动重命名。系统文档包括 `AI 使用指南/AI Guide`、`工作空间索引/Workspace Index`、`关于 SiYuan Agent Bridge/About SiYuan Agent Bridge`、`隐私规则/Privacy Rules`。不要新增 `siyuan_write_index`；Workspace Index 继续由现有 `siyuan_create_document` 和 `siyuan_edit_document` 维护。Privacy Rules 使用 Markdown 表格，不使用 JSON；模板表头优先使用 `Hide / Notebook ID / Notebook Name / Reason` 和 `Hide / Document ID / Title / Reason`，同时兼容中文表头、旧表头、空格和括号差异。文档隐藏必须用 Document ID；笔记本优先用 ID，没有 ID 时用名称，重名则全部隐藏。不做 `siyuan.ignore.local.json` / `siyuan.allow.local.json` fallback 或合并，Privacy Rules 是唯一主副本。Privacy Rules 是 MCP 内部配置，AI 不可读取、搜索、总结或编辑。实现后更新 Skill、README、相关文档和测试，最后运行 `pytest tests/ -v`。
