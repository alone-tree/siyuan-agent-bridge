SiYuan Bridge 架构图

给协作者快速理解程序分层、调用关系和数据边界。完整契约和实现细节以 docs/ARCHITECTURE.md 为准；整体架构大改时，本页必须同步。

事实基准：2026-06-07；MCP server 版本 1.0.0；实际暴露 9 个 MCP 工具。

先看这三件事

  1. 思源插件只负责安装、配置和反馈入口；真正的工具能力在 Python MCP Bridge。
  2. AI 客户端通过 MCP 调 Bridge；仓库里的 Skill 只是副本，必须被 AI 客户端主动导入后才生效。
  3. Privacy Rules 只由 MCP 内部解析，AI 不可读、不可搜、不可编辑。

主要调用链

场景调用关系
首次使用思源插件读取工作空间 Token → 写入 bridge/config.local.json → 用户复制 MCP JSON 到 AI 客户端。
Skill 生效仓库提供 Skill 副本 → 用户或 AI 客户端主动导入 → AI 才会按 Skill 约束调用工具。
会话启动siyuan_start → 探测 profile → 确保系统笔记本 → 解析 Privacy Rules → 刷新安全索引 → 返回启动包。
搜索siyuan_find → 临时打开笔记本 → 思源搜索/SQL → 隐私过滤 → 按文档聚合。
阅读siyuan_read → 解析可见文档 → getChildBlocks → 块窗口 + 大纲 → 提取附件。
写入siyuan_create/edit/doc_manage → confirmed=true + 权限检查 → 快照 → 写思源 → 路径同步 → 安全刷新索引。
