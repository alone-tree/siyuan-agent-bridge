# 思源桥：让顶尖 AI Agent 读懂你的思源笔记

AI 很强，但它不认识你。

你的研究资料、读书笔记、日记随笔、文章草稿、投资体系，可能已经在思源里积累了几年。可每次打开 Claude Code、Codex 或其他 AI Agent，它仍然像第一次见你一样：不知道你的资料在哪里，不知道哪些文档重要，不知道你长期形成的判断和偏好。

思源桥要解决的就是这件事：**把思源笔记接入 AI Agent，让你的本地笔记成为 AI 可以导航、搜索、阅读和维护的结构化个人知识库。**

它不是在思源里塞一个聊天机器人。它是反过来，把思源接到你已经在使用的顶尖 AI Agent 上。

## 为什么是“桥”

笔记软件内置 AI 很方便，但很难持续追上专门为 Agent 工作流设计的工具。Claude Code、Codex、Cursor 这类工具更新很快，已经拥有网络搜索、代码执行、文件解析、子 Agent 调度等能力。

与其等待每个笔记软件慢慢接入这些能力，不如让笔记成为 AI Agent 的知识层。

思源桥就是中间这座桥：

- 思源继续负责长期、本地、结构化地保存你的知识。
- AI Agent 负责理解、推理、搜索、写作、分析和执行任务。
- 思源桥负责把两边安全、稳定、低心智负担地连接起来。

这样你可以继续用自己熟悉的思源写笔记，同时让最新的 AI Agent 直接使用这些笔记。

## 像操作代码库一样操作知识库

AI 编程工具之所以好用，不只是模型聪明，也因为工具设计得好。AI 熟悉 `list / search / read / edit / write` 这一套工作方式。

思源桥没有把几十个思源底层 API 原样丢给 AI 猜，而是把它们收束成接近编程工具心智的 MCP 工具：

- `siyuan_list`：列出可见笔记本和文档层级。
- `siyuan_find`：搜索你的知识库。
- `siyuan_read`：按大纲和块窗口分段阅读长文档。
- `siyuan_edit`：基于块 ID 精确编辑。
- `siyuan_create`：创建或重写文档。
- `siyuan_doc_manage`：管理文档树，例如改名、移动、复制、导出。

AI 不需要理解一堆陌生 API。它只需要像探索代码项目一样探索你的笔记：先看目录，再搜索，再阅读，再在你确认后修改。

## 不做向量化碎片检索

很多知识库工具会把文档切碎、向量化，再把相关片段交给 AI。这样很快，但 AI 拿到的往往只是脱离上下文的几段文字。

思源桥选择另一条路：目录、全文搜索、大纲、分段阅读。

它会让 AI 先知道整个工作空间里有什么，再进入具体笔记本和文档。阅读长文档时，先返回大纲，再按块窗口返回正文。AI 始终知道自己读到的是哪一节、哪一段，而不是只拿到几个孤立片段。

慢一点，但更像人认真查资料，也更适合需要准确理解上下文的工作。

## 工作空间索引：让 AI 第一次来就知道你有什么

思源桥会在思源中维护一个“工作空间索引”。

你可以让 AI 扫描所有笔记本，为整个工作空间生成一份导航索引：每个笔记本是做什么的，哪些路径重要，关键文档在哪里，什么主题应该先去哪里找。

之后每次新会话，AI 不用从零开始猜。它启动时就能拿到这份索引，快速判断：

- 投资研究资料在哪里。
- 写作素材在哪里。
- 日记和想法在哪里。
- 哪些文档是入口，哪些只是细节。

这相当于给你的个人知识库加了一份 README，而且这份 README 可以由 AI 帮你维护，也可以由你在思源里直接修改。

## 记忆和规则，放回你自己的笔记里

很多 AI 工具也有记忆文件，但它们常常藏在系统目录里。对非程序员来说，不好找，不好改，也不直观。

思源桥把这些长期规则放回思源。

启用后，它会在思源中创建一个系统笔记本，里面有：

- **AI 使用指南**：你希望 AI 遵守的长期偏好、写作风格、重点笔记本。
- **工作空间索引**：整个知识库的语义导航。
- **隐私规则**：哪些笔记隐藏，哪些只读，哪些可写。
- **关于思源桥**：给人看的说明。

你不用去找配置文件，也不用懂 JSON。直接在思源里编辑这些文档即可。你的笔记软件就是管理 AI 的控制台。

## 隐私不是靠提示词，而是靠机制

思源桥的隐私规则由你在思源中维护。

你可以把某个笔记本或文档设置为：

- 隐藏：AI 看不到、搜不到、读不到、也不能写。
- 只读：AI 可以读取、复制或导出，但不能修改、移动、重命名或删除。
- 读写：AI 可以在你明确确认后写入。

关键是：隐私规则文档本身也被系统隔离。AI 不能读取它，不能搜索它，也不能修改它。

这不是要求 AI “自觉不要看”。这是从代码层面切断访问路径。

## 写入前先快照

AI 可以帮你写，也可能写错。

因此思源桥的写入工具都很克制。创建、编辑、移动、删除等写入操作，必须由用户明确要求，并且需要 `confirmed=true`。真正写入前，思源桥会先创建思源工作空间快照。

快照成功，写入才会继续。快照失败，直接拒绝。

如果结果不满意，你可以通过思源原生快照手动恢复。

## 适合什么场景

思源桥适合已经在用思源沉淀长期资料，并且也在使用 AI Agent 的人。

典型场景：

- 让 AI 从你的投资研究笔记中查找资料、整理逻辑、辅助写分析。
- 让 AI 阅读你的文章草稿、历史日记和思考记录，延续你的表达方式。
- 让 AI 在几百篇收藏文章中找到相关内容，而不是让收藏夹继续吃灰。
- 让 AI 根据你的工作空间索引快速定位资料，不用每次重新解释背景。
- 让 AI 在你确认后维护文档、追加内容、整理表格、导出 Markdown。

## 它不是什么

思源桥不是思源内置聊天助手。

它也不是“把所有思源 API 都暴露给 AI”的万能面板。

它更像一个面向 AI Agent 的知识库接口：把思源长期积累的内容，以 AI 能稳定使用的方式交给外部 Agent，同时保留本地、隐私、快照和人工控制。

## 使用前提

- 需要思源桌面端。
- 需要本机可用 Python。
- 需要 Claude Code、Codex 或其他支持 MCP 的 AI Agent。
- 使用时需要保持对应思源工作空间运行。

插件会生成 MCP 配置，你复制到 AI Agent 中即可使用。Token 保存在本机插件配置里，不写入 MCP JSON。

## 一句话

思源桥让你的思源笔记不再只是资料库，而是 AI Agent 可以真正使用的个人知识系统。

不是让笔记变多，而是让已有的笔记重新被看见、被理解、被使用。
