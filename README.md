# 思源桥

让 AI agent 安全地读取、搜索和编辑你的思源笔记。

---

## 核心亮点

### 高效：贴近 AI 编程心智

核心工具对标 AI 编程工具，心智模型对齐编程的同时，保留思源笔记的块结构设计。

| 工具 | 功能 | 类比 |
|------|------|------|
| `siyuan_start` | 自动注入笔记索引和用户偏好 | `CLAUDE.md` |
| `siyuan_list` | 列出笔记本和文档树 | `ls` |
| `siyuan_find` | 搜索知识库 | `grep` |
| `siyuan_read` | 分段阅读，含大纲导航和块 ID 引用 | `read` |
| `siyuan_edit` | 文档编辑：支持单块和多块、表格坐标编辑 | `edit` |
| `siyuan_create` | 创建或重写文档 | `write` |
| `siyuan_doc_manage` | 改名、移动、删除、复制、导出 | 文件管理 |

每个工具内部做了大量封装——在确保精准操作的同时，尽可能降低出错可能。让 AI 操作笔记时，尽可能接近修改文档和编程的体验，同时尊重思源的块结构设计。

不做大而全的 MCP，只专注打磨最高频的核心工具。

### 人类参与

隐私规则和AI指南全部在思源笔记内直接维护，不需要在别处找配置文件：

- **隐私规则**：在笔记本里直接编辑隐私规则文档，修改后让AI”刷新一下”即可生效。AI 无法感知更无法修改。注意：关闭的笔记本不代表对 AI 隐藏——它们会被临时打开以检索和读取，用完再关闭。如果你想让 AI 完全看不到某个笔记本，请将其加入隐私规则的隐藏列表，或设为只读。
- **AI 使用指南**：直接在思源里告诉 AI 你的偏好和规则，随时可改、随时生效。
- **工作空间索引**：AI 生成的笔记本导航索引，你可以参与修改、补充标注，让 AI 更准确地定位知识。

人始终在场。

### 稳定省心

- **MCP 与思源完全解绑**：无论思源笔记是否启动，MCP 都能注册成功。调用过程中随时可以关闭思源，需要时再打开即可——不会因为思源离线导致 MCP 注册失败或工具长时间卡住。
- **零依赖**：只使用 Python 标准库，不依赖任何第三方 package。有 Python 3.11+ 就能用。
- **一键配置**：插件自动读取工作空间 Token，生成 MCP JSON，复制给AI，让它”安装一下”即可使用。
- **权限继承**：隐私规则的权限向下继承——父文档设为只读，子文档自动只读；父文档隐藏，整棵子树全部隐藏。删除子文档时也会向上检查，父文档只读则子文档不可删。如需修改只读内容，AI 可以将其复制到有读写权限的位置再编辑。你也可以在隐私规则里面将一片文档权限临时改为”读写”以开放权限，用完再改回去。
- **关闭笔记本也能查**：关闭的笔记本代表过去积累的知识——只是最近不常用，不等于没有价值。思源桥会透明地临时打开关闭的笔记本进行检索和读取，用完再关回去。如果想让 AI 完全看不到某个笔记本，请将其加入隐私规则的隐藏列表，或设为只读。


## 使用场景

**让 AI 帮你整理知识库**：AI 读完你的笔记后，可以帮你重写文档、补充标签、整理目录结构——就像重构代码一样重构笔记。

**跨文档引用和总结**：AI 同时读多篇相关笔记，给你一个综合性的答案，并附带每个核心证据的块索引，而不只给一个文档标题。



## 安装

1. 在思源集市搜索「思源桥」安装插件
2. 插件设置页保存工作空间 Token
3. 复制生成的 MCP JSON 到 AI 客户端
4. 重启 AI 客户端，说"帮我查一下笔记里的 XXX"

**需要准备**：Python 3.11+、思源笔记。



## 当前不支持

- **手机端** — 本地Python程序，不适用移动端
- **数据库（Database）** — 只读渲染为普通表格，暂不支持编辑
- **闪卡、标签、块样式设置等功能** — 非知识库编辑核心功能，后续有需求再增加



## 常见问题

- **是否支持多个思源空间**：支持。不需要在每个工作空间都下载一个插件并注册，否则 AI 会看到两个完全相同的 MCP 工具。将插件安装到主要使用的工作空间，手动将其他工作空间的 Token 加入插件即可。隐私规则、AI 指南等存放在各自的工作空间，自动跟随切换。注意：每次只能启动一个思源空间。切换流程：1）打开目标思源空间，2）关闭其他工作空间，3）关闭目标思源空间，4）再次启动思源笔记。之所以需要 4 步，是因为思源默认首个启动的空间使用固定端口，后续空间使用随机端口，程序难以自动探测。
- **提示思源未启动**：先打开思源桌面端，确认工作空间正确。如果思源已启动仍报错，可能是 AI agent 或网络代理问题，请重启 AI agent。目前发现 Claude Code 在特定情况下 MCP 功能会出现 bug（例如启用 VPN 时打开 Claude Code，之后关闭 VPN）。
- **配置后 AI 看不到工具**：确认 MCP JSON 已复制到对应客户端且格式正确，并重启客户端。不同 AI agent 的 MCP 注册格式有细微差异，目前只提供 Claude Code 版本，后续会增加适配。你也可以让 AI 帮你手动调整。
- **会上传笔记内容吗？** 不会。用户体验改进计划只记录版本信息、工具名、耗时、调用是否成功、错误类型，用于判断高频但易出错的功能，便于针对性优化。不会上传任何笔记内容或对话内容。该功能默认关闭，你也可以选择仅本地存储供自己分析。
- **修改、写入后内容消失/产生冲突/崩溃**：这是由于两台电脑同时打开同一个思源空间，且开启了自动云同步。请将自动同步改为手动同步（仅启动、关闭时同步）。
- **修改、写入后没有产生快照**：同样是自动云同步的原因。在云同步时会自动生成快照，导致某些情况下，编辑前生成快照被检测为“无变化，跳过生成快照”。请将自动同步改为手动同步（仅启动、关闭时同步）。
- **快照会膨胀吗？**：思源内置快照自动清理（每天保留 2 个，180 天后删除），但需开启云同步才会触发。如果使用本地工作空间且未开启云同步，建议定期去「设置 → 数据仓库 → 清理」手动清理。快照清理功能正在开发中。
- **siyuan_doc_manage 能做什么？**：对标思源笔记原生的文档管理功能。改名、复制、导出只针对单篇文档；移动和删除则作用于整棵文档子树（文档及其所有子文档），与思源内部的操作逻辑一致。
