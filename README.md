# SiYuan Agent Bridge

将思源笔记接入 AI agent，作为 AI 的结构化知识库。

---

## 这是什么

思源代理桥（简称"思源桥"）是一个 MCP 适配层，让 Claude Code、Codex、Cursor 等 AI agent 能安全地读取、搜索和写入你的思源笔记。

它不是思源插件，而是AI插件。你的笔记仍然保存在思源里。思源桥只负责生成安全索引，把笔记变成 AI 可以理解的结构化上下文。

## 为什么需要它

AI 能力很强，但 AI 记不住你。网页版 AI 有记忆功能，但凌乱、碎片化、容量有限。AI 编程工具提供了记忆文件，但管理不便。

思源桥让 AI 可以直接"读懂"你在思源里积累的所有笔记——研究资料、日记、文章草稿、读书笔记——然后给你更精准的帮助。它不是把 AI 塞进笔记软件，而是把你的笔记接入最顶尖的 AI agent。

**以 AI 为中心，而非以笔记为中心。** 当 AI 工具更新时，你可以立刻用上最新能力，不必等待笔记软件开发者升级支持。

## 核心理念

### 桥接，而非内置

Notion、Obsidian、思源等工具都在尝试内置 AI。但笔记软件内置 AI 很难追上 Claude Code、Codex 这类专门为 agent 工作流设计的工具。桥接层更灵活：AI 工具一更新，思源桥就能跟上。

### 像编程一样处理文档

AI 编程工具之所以强，不只是因为 AI 聪明，更因为工具设计得好。思源桥借鉴了这个思路，把丰富的思源 API 封装为 5 个易于 AI 使用的核心工具：

| 工具 | 功能 | 类比 |
|------|------|------|
| `siyuan_list` | 列出笔记本和文档树 | `ls` / 文件浏览器 |
| `siyuan_find_documents` | 搜索知识库 | `grep` |
| `siyuan_read_document` | 分段阅读文档（含大纲导航） | `read` with line ranges |
| `siyuan_edit_document` | 文本替换编辑 | `edit` |
| `siyuan_create_document` | 创建新文档 | `write` |

每个工具都不只是简单暴露一个 API，而是将多个接口整合封装，减少 AI 出错的可能。

阅读时，思源桥会先提取文档大纲，标注每节标题的块位置，再按窗口分段返回——就像编程工具按行号读文件一样，但更有逻辑性。

### 隐私优先

隐私规则完全由你在思源中维护。你可以将指定的笔记本或文档标记为隐藏，AI 就不会检索、读取、编辑它们。

隐私规则文档本身通过系统硬编码隔离——AI 无法感知它的存在，更无法修改。

### 人类始终参与

思源桥不会因为有了 AI 而减损人的任何能力。它会在思源中自动创建一个系统笔记本，包含：

- **AI 使用指南** — 你可以随时编辑，告诉 AI 你的偏好和规则
- **隐私规则** — 管理哪些内容对 AI 隐藏
- **工作空间索引** — AI 生成的全局导航，每次启动时自动加载
- **关于思源代理桥** — 简要使用说明

所有这些你都可以在思源里像编辑普通文档一样修改。AI 能做什么、不能做什么，由你来定。

### 工作空间索引

思源桥会提示 AI 阅览所有笔记本，生成一个索引摘要。之后每次新会话，AI 启动时就能立刻了解你所有笔记本的概况，快速定位目标，不用每次都从头扫描。

这不是简单的文档树打印，也不是无差别的全文摘要——AI 会判断每个笔记本的定位和重点，形成结构化的导航信息。

### 精确引用

思源桥可以在返回文档时附带每一段的块 ID，AI 引用时可以精确到具体段落，而不只是整篇文档。配合分段阅读，长文档也能精准导航。

### 写入安全

每次 AI 编辑前，思源桥都会先创建思源数据快照。快照写入成功，编辑才能生效。你可以随时通过思源的快照功能手动回滚到编辑前的状态。

## 安装

告诉你的 AI agent：

> 在我的xx文件夹有一个压缩包，请按照里面的指引，把程序安装到我的电脑上。

AI 会自动完成解压、配置、诊断和 MCP 注册。安装完成后重启 AI 客户端，在新会话中说"帮我查笔记"即可。

**需要准备**：Python 3.11+、思源笔记运行中、思源 API Token（思源 → 设置 → 关于 → 复制）。

---

## 当前不支持

- **Mac / Linux** — 当前仅 Windows。因为我没有其他系统的电脑，欢迎提交 PR。
- **手机端** — 思源桥是本地 Python 程序，依赖本地 MCP，暂不支持移动端。
- **团队协作** — 思源本身是面向个人的笔记软件，思源桥默认为单人使用场景。
- **数据库（Database）** — 暂简化为普通表格，不提供数据库的操作接口。
- **超级块嵌套布局** — 展平为 Markdown，不保留复杂排版结构。

## 项目结构

```
siyuan-agent-bridge/
├── source_code/                  # Python 适配层
├── plugins/siyuan-agent-bridge/  # MCP 插件（skills + 启动脚本）
├── mcp_configs/                  # 多平台 MCP 配置模板
├── knowledge_base/               # 运行时生成的索引（Git 忽略）
├── ai_workspace/                 # AI 工作区（附件、分析、草稿）
├── config.example.json           # 配置示例
├── doctor.bat                    # 诊断脚本
├── INSTALL_FOR_AI.md             # AI 安装指南（AI 读这个）
└── README.md                     # 本文件（给人看）
```

更多工程细节考量请见[PRO文档](docs/PRO.md)。
