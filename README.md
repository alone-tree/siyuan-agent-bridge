# SiYuan Bridge

将思源笔记接入 AI agent，作为 AI 的结构化知识库。

---

## 0.3 更新

当前版本聚焦遥测与反馈系统：新增 `siyuan_bridge_feedback` 工具，遥测基础框架和代理支持。

- 新增 `siyuan_bridge_feedback` 工具：AI 可直接通过对话提交 bug 报告、功能请求或想法，无需依赖思源。
- 遥测框架：可选的匿名使用统计，帮助开发者了解工具使用情况和性能瓶颈。默认关闭，仅本地存储，不收集内容。
- 代理支持：自动探测系统代理或环境变量，国内用户通过 VPN 可正常使用遥测上传和反馈功能。

当前版本聚焦两个能力：更稳定的结构化编辑，以及更精简的 MCP 工具面。

- 工具名更清晰：`siyuan_find`、`siyuan_read`、`siyuan_create`、`siyuan_edit`、`siyuan_doc_manage`。
- 编辑更稳：先用 `siyuan_read(include_block_ids=true)` 获取块序号和块 ID，再用 `siyuan_edit` 精确修改。
- 表格更友好：普通 Markdown 表格会以带行列号的网格视图呈现，AI 可以按 `row` 和 `column_index` 修改单元格、插入/删除行列。
- 写入更安全：创建、编辑和文档管理写操作前都会创建思源工作空间快照，隐藏文档不可读也不可写。

## 这是什么

思源桥（SiYuan Bridge）是一个 MCP 适配层，让 Claude Code、Codex、Cursor 等 AI agent 能安全地读取、搜索和写入你的思源笔记。

> **项目原名 SiYuan Agent Bridge**，于 v0.2 更名为 SiYuan Bridge（思源桥）。如果你之前按旧名配置过，请按新版安装指引重新配置 MCP。

它的核心是本地 Python MCP Bridge，思源插件负责安装、配置和生成 MCP JSON。你的笔记仍然保存在思源里，思源桥只负责把笔记变成 AI 可以安全使用的结构化上下文。

## 为什么需要它

AI 能力很强，但 AI 记不住你。网页版 AI 有记忆功能，但凌乱、碎片化、容量有限。AI 编程工具提供了记忆文件，但管理不便。

思源桥让 AI 可以直接"读懂"你在思源里积累的所有笔记——研究资料、日记、文章草稿、读书笔记——然后给你更精准的帮助。它不是把 AI 塞进笔记软件，而是把你的笔记接入最顶尖的 AI agent。

**以 AI 为中心，而非以笔记为中心。** 当 AI 工具更新时，你可以立刻用上最新能力，不必等待笔记软件开发者升级支持。

## 核心理念

### 桥接，而非内置

Notion、Obsidian、思源等工具都在尝试内置 AI。但笔记软件内置 AI 很难追上 Claude Code、Codex 这类专门为 agent 工作流设计的工具。桥接层更灵活：AI 工具一更新，思源桥就能跟上。

### 像编程一样处理文档

AI 编程工具之所以强，不只是因为 AI 聪明，更因为工具设计得好。思源桥借鉴了这个思路，把丰富的思源 API 封装为几个易于 AI 使用的核心工具：

| 工具 | 功能 | 类比 |
|------|------|------|
| `siyuan_list` | 列出笔记本和文档树 | `ls` / 文件浏览器 |
| `siyuan_find` | 搜索知识库 | `grep` |
| `siyuan_read` | 分段阅读文档（含大纲导航） | `read` with line ranges |
| `siyuan_edit` | 引用阅读定位后的结构化编辑 | `edit` |
| `siyuan_create` | 创建或重写文档 | `write` |
| `siyuan_doc_manage` | 改名、移动、删除、复制、导出文档 | 文件管理 |
| `siyuan_bridge_feedback` | 通过对话提交反馈和建议 | 反馈 |

另有两个辅助工具：`siyuan_start` 用于启动和刷新安全上下文，`siyuan_refresh_index` 用于手动刷新索引。

每个工具都不只是简单暴露一个 API，而是将多个接口整合封装，减少 AI 出错的可能。

阅读时，思源桥会先提取文档大纲，标注每节标题的块位置，再按窗口分段返回——就像编程工具按行号读文件一样，但更有逻辑性。

编辑时，AI 会先进入引用阅读模式，拿到形如 `[12] id=... type=paragraph` 的定位信息，再用 `siyuan_edit` 修改。普通 Markdown 表格会显示为带 `row` 和 `column_index` 的网格，AI 不需要猜 Markdown 分隔行，也不需要把表格当数据库处理。

### 隐私优先

隐私规则完全由你在思源中维护。你可以将指定的笔记本或文档标记为隐藏，AI 就不会检索、读取、编辑它们。

隐私规则文档本身通过系统硬编码隔离——AI 无法感知它的存在，更无法修改。

### 人类始终参与

思源桥不会因为有了 AI 而减损人的任何能力。它会在思源中自动创建一个系统笔记本，包含：

- **AI 使用指南** — 你可以随时编辑，告诉 AI 你的偏好和规则
- **隐私规则** — 管理哪些内容对 AI 隐藏
- **工作空间索引** — AI 生成的全局导航，每次启动时自动加载
- **关于思源桥** / **About SiYuan Bridge** — 简要使用说明

所有这些你都可以在思源里像编辑普通文档一样修改。AI 能做什么、不能做什么，由你来定。

### 工作空间索引

思源桥会提示 AI 阅览所有笔记本，生成一个索引摘要。之后每次新会话，AI 启动时就能立刻了解你所有笔记本的概况，快速定位目标，不用每次都从头扫描。

这不是简单的文档树打印，也不是无差别的全文摘要——AI 会判断每个笔记本的定位和重点，形成结构化的导航信息。

### 精确引用

思源桥可以在返回文档时附带每一段的块 ID，AI 引用时可以精确到具体段落，而不只是整篇文档。配合分段阅读，长文档也能精准导航。

### 写入安全

每次 AI 编辑前，思源桥都会先创建思源数据快照。快照写入成功，编辑才能生效。你可以随时通过思源的快照功能手动回滚到编辑前的状态。

`siyuan_create` 优先使用完整可读路径，例如 `/Notebook/Folder/Doc`，程序内部会自动解析笔记本 ID 和思源内部路径。目标文档已存在时默认拒绝写入；也可以显式选择覆盖现有文档（清空块后重写，保留文档 ID）或新增一个同名文档。写入后会等待思源路径同步并自动刷新安全索引。

`siyuan_list` 会显示每个可见笔记本或文档的有效权限：`read_write` 可写，`read_only` 只能读取、复制或导出。隐藏内容不会出现在列表里。

`siyuan_doc_manage` 处理文档树管理：`rename/move/delete/copy/export`。`copy/export` 允许只读源文档；`rename/move/delete` 需要可写权限和 `confirmed=true`。`delete` 会删除整棵子树，因此子孙文档也必须全部可写；`move` 会保留子树，但会拒绝从只读/隐藏祖先下移出文档。`copy` 需要明确 `target_path`，使用思源复制接口复制单篇文档。除 export 外，操作后会等待路径同步并自动刷新索引。

## 安装

1. 在思源集市搜索「思源桥」安装插件。
2. 在插件设置页中配置你的思源 API Token。
3. 根据设置页显示的 MCP 配置（`command` + `args`），在你的 AI 客户端中注册 MCP 服务器。
4. 重启 AI 客户端，在新会话中说"帮我查笔记"即可。

**需要准备**：Python 3.11+、思源笔记运行中、思源 API Token（思源 → 设置 → 关于 → 复制）。

## 当前不支持

- **Mac / Linux** — 当前仅 Windows。因为我没有其他系统的电脑，欢迎提交 PR。
- **手机端** — 思源桥是本地 Python 程序，依赖本地 MCP，暂不支持移动端。
- **团队协作** — 思源本身是面向个人的笔记软件，思源桥默认为单人使用场景。
- **数据库（Database）** — 暂简化为普通表格，不提供数据库的操作接口。
- **超级块嵌套布局** — 展平为 Markdown，不保留复杂排版结构。

## 常见问题

- **工具提示思源未启动**：先打开思源桌面端，确认当前工作空间正确，再检查插件里的 API Token。
- **所有 MCP 工具突然超时**：如果刚切换过 VPN/代理，完全重启 AI 客户端后再试。
- **反馈或遥测发不出去**：不影响笔记功能；国内网络下可开代理，或直接关闭用户体验改进。
- **会上传笔记内容吗？** 不会。遥测只记录工具名、耗时、成功失败、版本和匿名 ID。
- **配置后 AI 还是看不到工具**：确认 MCP JSON 已复制到对应 AI 客户端，并重启客户端。
